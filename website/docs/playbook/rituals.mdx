# Rituals

IDD has fewer, more purposeful rituals than Scrum. Each one earns its place.

| Ritual | Cadence | Purpose | Output |
|---|---|---|---|
| **Intent Review** | 2x / week | Review new intent docs as a team. Challenge assumptions, sharpen acceptance criteria, catch missing context. | Approved intent queue with clear priority order |
| **Decision Digest** | Daily (async) | Each person posts their decisions from the day with one-line rationales in a shared channel. | Running decision log — the real project history |
| **Taste Sync** | Weekly | Review recent outputs holistically. Are we drifting? Are patterns emerging that should become principles? | Updated design principles + taste guidelines |
| **Context Refresh** | Weekly | Update CLAUDE.md and project context. Prune stale info. Add new conventions. | Fresh context files that make AI outputs better |
| **Direction Check** | Biweekly | Zoom out. Are we building the right thing? How has our understanding changed? What should we stop doing? | Updated roadmap priorities + parked intents |
