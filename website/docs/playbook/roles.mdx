# Roles

IDD doesn't have rigid roles. It has responsibilities that shift based on context. In a team of 2–10, everyone wears multiple hats. But three archetypes cover most of the work:

| Archetype | Core Responsibility | Not Responsible For |
|---|---|---|
| **Intent Architect** | Defining what to build and why. Writes intent docs. Owns the "what" and "why." Holds strategic context. | Writing code. Managing timelines. Approving PRs for code correctness. |
| **Taste Keeper** | Evaluating outputs against user experience, API design, and community standards. The quality bar. | Generating code. Sprint planning. Writing specs. |
| **Context Curator** | Maintaining the project's institutional memory: CLAUDE.md, decision logs, design principles, intent templates. | Making product decisions. Blocking releases. |

In a 3-person team, each person owns one archetype. In a 7-person team, some archetypes are shared. Everyone can run the core loop.
