## Week 1: Foundation

- Write your first CLAUDE.md with project conventions, patterns, and anti-patterns
- Create your intent document template (use the structure from the Phase Details page)
- Set up a shared decision log (a markdown file, a Notion page, a GitHub discussion — low friction wins)
- Run your first Intent Review with the team

## Week 2: Practice

- Run 3–5 complete IDD loops on real work
- Start the daily async Decision Digest
- Hold your first Taste Sync — identify 3 design principles from existing decisions
- Notice what's awkward. That's useful signal.

## Week 3: Calibrate

- Review convergence rates. Are intents clear enough? Are you redirecting too much or too little?
- Refine the intent template based on what's been missing
- Update CLAUDE.md with everything you've learned
- Drop any ritual that isn't earning its keep

## Week 4: Own It

- The framework is now yours to modify. Keep what works, cut what doesn't
- Establish your Convergence Dashboard
- Run your first Direction Check
- Write up your team's version of this framework — the one that actually matches how you work
